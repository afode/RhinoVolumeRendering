Features:

My intentions for this 3D renderer was to create an online tool that would be useful to doctors and their patients, as well as medical students.

Custom Layers:
Originally, I wanted to allow users to upload their own layers. I was able to upload my own files, but then had trouble retrieving the image data to import into my textureLayers() method. This is something I will continue to work on.

Controls GUI:
I am still working on functionality of my controls GUI, but I want to allow my users to "hide layers", as to get a better view throughout the volume and see each individual slice. I also want my users to be able to increase/decrease the transparency and brightness. The transparency control changes the transparency of all of the layers. The Brightness will increase/decrease the intensity of the lights. Lastly, I want to add the functionality to change the color of the lights, as to change the color of the volume.

Programming Tools and Languages:

For this project I used HTML and CSS along with Typescript and Babylon.js Library.

Typescript is a superset of Javascript which provides optional static typing, classes, and interfaces. After using it, I learned that it is more readable than JavaScript and, overall, it is more intuitive and user friendly. It reminds me a bit of C++ without the mess of pointers.

Babylon.js WebGL, or Web Graphics Library, is a Javascript API designed to render interactive 3D computer graphics and 2D graphics within any compatible web browser, without the use of any plug-ins. I used this API primarily for this project. It was simple and powerful and I recommend it for people interested in building simple online games. A similar WebGL is Three.js, which seemed a bit more powerful, having more features than Babylon. I will most likely use Three.js in the future. I am also thinking of re-writing my Rhino Volume Renderer using Three.js to see if it is any better in performance.
